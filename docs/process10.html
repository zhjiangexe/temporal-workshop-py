Activity用途
long_running_act在迴圈中定期 heartbeat，讓 Activity 能收到取消訊號。

workflow.py

@workflow.defn
class LongActWorkflow:
    @workflow.run
    async def run(self) -> None:
        await workflow.execute_activity(
            long_running_act,
            start_to_close_timeout=timedelta(seconds=30),
        )

activities.py — Heartbeat 觸發取消偵測

@activity.defn
async def long_running_act() -> str:
    for i in range(1000):
        activity.heartbeat(i)        # 這裡會檢查取消！
        await asyncio.sleep(0.01)
    return "DONE"
Heartbeat 的隱藏功能：取消偵測
activity.heartbeat() 不只是回報「我還活著」。呼叫 heartbeat 時，SDK 會向 Temporal Server 確認 Activity 的狀態。如果 Workflow 已經被取消（或 Activity 被 cancel），heartbeat() 會在 Activity 中拋出 CancelledError。

這就是為什麼長時間 Activity 應該定期呼叫 heartbeat——不只是為了偵測 Worker 死亡，也是為了及時響應取消請求。
如果 Activity 不呼叫 heartbeat？
如果一個長時間 Activity 從不呼叫 heartbeat，取消請求會被延遲到 Activity 自然完成或 timeout。Activity 不會被「強制殺死」。
只有在 await 點（如 asyncio.sleep）或 activity.heartbeat() 呼叫時，Activity 才有機會收到 CancelledError。

取消流程全貌

Client → cancel() → Temporal Server → 標記 Workflow 為 cancel requested
                                      ↓
                          Workflow 收到 CancelledError
                                      ↓
                          正在執行的 Activity 被標記為 cancel
                                      ↓
                          Activity 下次呼叫 heartbeat() 或 await 時
                          收到 CancelledError
                                      ↓
                          Activity 結束 → Workflow 結束
                          狀態: Cancelled

注意事項

asyncio.sleep 也會觸發取消
在 Python 的 asyncio 中，await asyncio.sleep() 是一個 cancellation point。當 Activity 被取消時，asyncio.sleep 會拋出 CancelledError。
所以即使不呼叫 heartbeat，只要有 await，Activity 也能收到取消。但 heartbeat 提供了更即時、更明確的取消管道。
在 Activity 中做 cleanup
如果 Activity 在取消時需要做清理（例如刪除暫存檔案），可以用 try/except： 
try:
    for i in range(1000):
        activity.heartbeat(i)
        await asyncio.sleep(0.01)
except asyncio.CancelledError:
    # 清理工作
    cleanup_temp_files()
    raise  # 仍然要 re-raise

延伸練習

  1. 在 Activity 中加入 try/except CancelledError，在取消時印出「已清理 X 筆暫存資料」
  2. 移除 activity.heartbeat()，只保留 asyncio.sleep，觀察取消是否仍然有效
  3. 把 asyncio.sleep(0.01) 換成一個同步的 time.sleep(0.01)，觀察取消行為的差異（提示：同步 sleep 不是 cancellation point）
