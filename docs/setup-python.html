這份 workshop 需要什麼？

工具本 workshop 用途確認指令
git下載這份 repository。git --version
uv建立 Python 環境、安裝 temporalio、執行測試與範例。uv --version
temporal啟動本機 Temporal Server 與 Web UI。temporal --version

建議安裝方式

macOS / Homebrew：

brew install uv temporal

如果沒有 Homebrew，至少要先依官方方式安裝 uv 與 temporal CLI。安裝完成後確認：

uv --version
temporal --version

下載並初始化本 workshop

git clone git@github.com:zhjiangexe/temporal-workshop-py.git
cd temporal-workshop-py
cd python
uv sync

uv sync 會根據 python/pyproject.toml 與 python/uv.lock 安裝 workshop 需要的 Python 套件。

確認 Python 版可以跑

cd python
uv run pytest

時間不夠時，先跑最小確認：

cd python
uv run pytest tests/process1 -q

啟動 Temporal Server

開一個 terminal，保持它執行：

temporal server start-dev
項目位置
Worker / Starter 連線位置localhost:7233
Web UIhttp://localhost:8233

跑第一個流程

開第二個 terminal 啟動 Worker：

cd python
uv run python -m temporal_learn.process1.worker

開第三個 terminal 啟動 Workflow：

cd python
uv run python -m temporal_learn.process1.starter
Workshop 原則
Python 範例一律用 uv run python -m ... 執行，避免用到錯誤的 Python 或缺少專案套件。

本 workshop 的常見狀況

狀況處理方式
Worker 連不上 Temporal先確認 temporal server start-dev 還在跑。
找不到 temporal_learn確認人在 python/ 目錄，並使用 uv run python -m ...。
想模擬 Process 2 上架失敗在 python/ 目錄執行 FAIL_PUBLISH=1 uv run python -m temporal_learn.process2_1.worker 或 process2_2.worker。
