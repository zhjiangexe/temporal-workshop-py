Process 7 — Update 與 Query

@workflow.update@workflow.queryState Mutation

學習目標

  • 學會用 @workflow.update 在 Workflow 執行期間修改內部狀態
  • 學會用 @workflow.query 即時讀取 Workflow 的內部狀態
  • 理解 Update 送入的時機如何影響最終結果

程式碼解說

閱讀順序：models → activities → workflow → worker → starter
  1. models.py：本流程未使用額外資料模型。
  2. activities.py：組合 greeting 字串。
  3. workflow.py：GreetingWorkflow 示範 Update 修改狀態、Query 讀取狀態。
  4. worker.py：本流程未提供獨立 worker 檔，可用測試或自行註冊。
  5. starter.py：本流程未提供獨立 starter，可用 Temporal CLI 或測試互動。

activities.py — Activity 清單

Activity用途
compose_greeting把目前 greeting 與 name 組合成最終文字。

workflow.py

@workflow.defn
class GreetingWorkflow:
    def __init__(self):
        self._current_greeting = "Hello"    # 可被 Update 修改的狀態

    @workflow.run
    async def greet(self, name: str) -> str:
        await asyncio.sleep(3600)           # 等 1 小時，給外部時間送 Update
        return await workflow.execute_activity(
            compose_greeting,
            args=[self._current_greeting, name],  # 用當時的 greeting
            start_to_close_timeout=timedelta(seconds=5),
        )

    @workflow.update
    async def update_greeting(self, greeting: str) -> None:
        self._current_greeting = greeting   # 修改內部狀態

    @workflow.query
    def get_greeting(self) -> str:
        return self._current_greeting       # 回傳目前狀態（唯讀）
Update 的時機很重要
這個範例展示了一個關鍵概念：Update 送入的時間點會影響最終結果。
如果在 asyncio.sleep(3600) 結束之前送了 Update，Activity 會用新的 greeting。
如果 sleep 結束之後才送（Workflow 已經完成），Update 會失敗。
Query 不佔 History 空間
@workflow.query 的呼叫不會記錄在 Event History 中。它直接在 Worker 的 Workflow 記憶體中執行，不經過 Temporal Server 的事件系統。
這表示你可以高頻呼叫 Query 而不用擔心 History 膨脹。

activities.py

@activity.defn
async def compose_greeting(greeting: str, name: str) -> str:
    return f"{greeting}, {name}!"

極簡 Activity，純粹組合字串。把它放在 Activity 而不是 Workflow 中，是因為在實際場景中這裡可能會做 IO（例如查資料庫取得使用者的顯示名稱）。

注意事項

Query handler 不能修改狀態
@workflow.query 方法必須是純讀取的。如果在 Query handler 中修改了 Workflow 的任何狀態，Temporal SDK 會拋出錯誤。
需要修改狀態 → 用 @workflow.update 或 @workflow.signal。
Query 的一致性
Query 回傳的是 Worker 記憶體中 Workflow 的當前快照。如果 Workflow 正在處理一個 Activity，Query 回傳的狀態是 Activity 開始前的狀態。
換句話說，Query 看到的是「最後一個 Workflow Task 完成後」的狀態。

延伸練習

  1. 加入一個 get_name Query 回傳 Workflow 的 name 參數
  2. 讓 update_greeting 回傳修改前的舊值（-> str）
  3. 加入一個 Update validator：只允許長度 ≤ 20 的 greeting
