Process 2-1 python/src/temporal_learn/process2_1/ 直接在 workflow.py 維護 compensations 清單，正向 Activity 執行前先登記安全補償。 初學 Temporal 時優先看這版，最貼近 Workflow 呼叫 Activity 的原始概念。

建議閱讀順序
先看 Process 2-1，理解 Temporal Workflow 如何直接編排正向 Activity 與補償 Activity；再看 Process 2-2，理解 helper 只是在整理程式碼，不是 Temporal SDK 的內建功能。

補償流程示範開關

Process 2-1 與 Process 2-2 都可以用 FAIL_PUBLISH=1 啟動 Worker， 模擬 publish_to_storefront 失敗，進而觀察補償 Activity 是否依反向順序執行。

版本模擬上架失敗啟動 Workflow
Process 2-1 cd python && FAIL_PUBLISH=1 uv run python -m temporal_learn.process2_1.worker cd python && uv run python -m temporal_learn.process2_1.starter
Process 2-2 cd python && FAIL_PUBLISH=1 uv run python -m temporal_learn.process2_2.worker cd python && uv run python -m temporal_learn.process2_2.starter
注意
FAIL_PUBLISH=1 要加在 Worker 啟動時，因為失敗邏輯是在 Activity 執行端判斷。 如果只加在 starter 上，Workflow 仍會送出，但 Activity 不會模擬失敗。
